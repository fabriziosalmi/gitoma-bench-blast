# Contributing to gitoma-bench-blast

Welcome! We welcome contributions from everyone. If you'd like to help improve this project, here is a guide on how to contribute.

## How to Contribute

Contributions can take many forms, and any help is greatly appreciated. Here are some ways you can contribute:

*   **Report Bugs:** If you find a bug, please report it clearly. Include steps to reproduce the issue and any relevant logs or environment details.
*   **Enhance Features:** If you have an idea for a new feature or improvement, please suggest it. If you are comfortable with coding, feel free to implement it!
*   **Documentation:** Improving documentation is always welcome. If you see something unclear or missing, please add to the `README.md` or this guide.

## Development Workflow

To contribute code, please follow these general steps:

1.  **Fork the Repository:** Fork our repository to your account.
2.  **Clone Your Fork:** Clone your fork locally.
3.  **Create a Branch:** Create a new branch for your feature or fix (e.g., `git checkout -b feature/my-new-thing`).
4.  **Make Changes:** Implement your changes, ensuring you keep all existing tests passing.
5.  **Test Thoroughly:** Run the test suite to verify your changes haven't introduced regressions.
6.  **Commit:** Commit your changes with a clear, descriptive commit message following [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).
7.  **Push and Pull Request:** Push your branch to your fork and open a Pull Request against the `main` branch of this repository.

## Code Style and Standards

*   **Language:** The project is primarily in Python.
*   **Style Guide:** Please adhere to [PEP 8](https://peps.iana.org/pep-0008/) guidelines for Python code.
*   **Testing:** All new features must include corresponding tests. Fixes should also ideally have a test case demonstrating the fix.

Thank you for your contribution!
