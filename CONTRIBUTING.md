# Contributing to gitoma-bench-blast

Welcome! If you wish to contribute to this project, we welcome your help. Whether it's finding bugs, suggesting features, or writing tests, your contributions are valuable.

## How to Contribute

1.  **Read the Docs:** Please read the `README.md` to understand the project's goals and current state.
2.  **Report Issues:** If you find a bug or have a feature request, please open an issue in the repository. Provide as much detail as possible, including steps to reproduce bugs.
3.  **Feature Requests:** If you have an idea for a new feature, please describe it in an issue. We will review it against the project roadmap.
4.  **Code Contributions:** If you wish to submit code changes:
    *   Fork the repository.
    *   Create a new branch for your feature or fix.
    *   Make your changes.
    *   Commit your changes with a clear, descriptive commit message (following [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/)).
    *   Push to your branch.
    *   Open a Pull Request against the main branch.

## Development Setup

To contribute effectively, please follow the setup instructions in the `README.md` to ensure you are running against the correct environment.

## Style Guide

We adhere to standard Python conventions. Please ensure your code follows PEP 8 guidelines. Linters and formatters are recommended for consistency.

## Testing

All contributions should include tests that verify the changes made. Please ensure your tests pass before submitting a Pull Request.

Thank you for contributing!
