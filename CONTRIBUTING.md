# Contributing to gitoma-bench-blast

Welcome! We love contributions. If you want to help improve this project, here is a guide on how to get started.

## How to Contribute

Contributions are welcome in many forms, but if you are looking to contribute code, please follow these steps:

1.  **Fork the repository:** Click the fork button on GitHub and create your own copy of the project.
2.  **Clone your fork:** Clone your newly forked repository to your local machine.
3.  **Create a new branch:** Create a descriptive branch for your changes (e.g., `git checkout -b feature/my-new-thing`).
4.  **Make your changes:** Implement the requested feature, fix the bug, or improve the documentation.
5.  **Test your changes:** Ensure all existing tests pass and that your new code works as intended.
6.  **Commit your changes:** Commit your work with a clear, descriptive commit message following [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).
7.  **Push to your branch:** Push the new branch to your fork.
8.  **Open a Pull Request (PR):** Open a Pull Request from your branch into the `main` branch of the original repository.

## Code Style and Standards

*   **Language:** The primary language is Python.
*   **Testing:** All new features must include corresponding tests. Please ensure your changes pass all existing unit and integration tests.
*   **Formatting:** We use [Black](https://github.com/psf/black) for automatic code formatting. Please run the formatter before committing.
*   **Linting:** We use [Ruff](https://docs.ruff.readthedocs.io/en/latest/) for linting. Please address all warnings before submitting a PR.

## Reporting Bugs

If you find a bug:
1.  Try to reproduce it locally.
2.  Provide the steps to reproduce (including environment details).
3.  If possible, include a minimal reproducible example.
4.  Report it in the Issues section with a clear title.

## Roadmap and Ideas

If you have an idea for a new feature:
1.  Check if the feature already exists.
2.  Describe the idea clearly in a new Issue labeled `enhancement` or `feature request`.

Thank you for your time and contribution!
