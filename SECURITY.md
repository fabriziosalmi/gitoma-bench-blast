# Vulnerability Disclosure Policy

We welcome security reports! If you find a security vulnerability in `gitoma-bench-blast`, please report it to us responsibly.

## Reporting a Vulnerability

Please follow these steps to report a security issue:

1. **Do not** disclose the vulnerability publicly.
2. Contact us privately via [Insert Responsible Disclosure Email Here].
3. Provide as much detail as possible about the vulnerability, including steps to reproduce it, but **do not** include sensitive information or exploit code in your initial report.

## Our Commitment

Upon receiving a report, we will:

*   Acknowledge receipt of the vulnerability report.
*   Assess the validity and severity of the issue.
*   Work with you to develop a fix in a timely manner.
*   Once the patch is ready, we will coordinate with you on its disclosure timeline to ensure users are protected.

We aim to address reported vulnerabilities promptly and maintain the security of our project.
