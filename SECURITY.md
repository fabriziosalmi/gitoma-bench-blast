# Security Policy

This project is committed to maintaining a secure codebase. We take security vulnerabilities seriously and encourage responsible disclosure.

## Responsible Vulnerability Disclosure

If you have found a security vulnerability in this project, please do not disclose it publicly. Instead, follow these steps to report it to us securely:

1. **Contact Us:** Please reach out to the maintainers via a private channel (e.g., email at security@[projectdomain].com or through a dedicated issue tracker if available).
2. **Provide Details:** Include as much detail about the vulnerability as possible, including steps to reproduce it, but **do not** include any sensitive information or exploit code that could be misused.
3. **Wait for Instructions:** We will assess the vulnerability and work with you to find a patch before making any public acknowledgment.

We commit to responding to all vulnerability reports in good faith and will keep the reporter informed throughout the remediation process.

## Security Policy Timeline

Once a vulnerability is reported, we aim to:

*   Acknowledge receipt of the report promptly.
*   Assess the severity and scope of the issue.
*   Develop and test a fix in private branches.
*   Release a patch or advisory once the fix is ready and vetted.

## Terms of Use

By reporting a vulnerability, you agree to abide by this disclosure policy. Any attempt to exploit vulnerabilities without prior permission from the maintainers will be considered malicious and may result in legal action.
